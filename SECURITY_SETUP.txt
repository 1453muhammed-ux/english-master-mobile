WORDPILOT v5.0 — GÜVENLİK KURULUM PLANI

GERÇEK: F12 TAMAMEN KAPATILAMAZ
Bir tarayıcı bir içeriği gösterebiliyorsa o içerik cihazın belleğine veya ağ katmanına gelir. Bu nedenle HTML, JavaScript, CSS ve statik JSON dosyalarını F12’den mutlak biçimde gizlemek mümkün değildir. Sağ tık/klavye engelleri yalnızca sıradan kopyalamayı caydırır; güvenlik duvarı değildir.

V5 PAKETİNDE HAZIR OLAN KORUMALAR
1. Firebase Hosting güvenlik başlıkları: CSP, frame engeli, nosniff, referrer ve izin politikaları.
2. Firebase App Check entegrasyon noktası.
3. Kullanıcı yalnızca kendi /users/{uid} belgesini okuyup yazabilir.
4. AI anahtarı ön yüzde değil Cloud Functions Secret içinde tutulur.
5. AI callable function giriş, App Check ve kullanıcı başına hız sınırı ister.
6. Ders alanlarında sağ tık, seçim ve toplu kopyalama caydırılır.
7. Site PWA olarak mağazasız kurulabilir; mağazaya çıkmak güvenlik için zorunlu değildir.

BİRLİKTE EKRAN GÖRÜNTÜLERİYLE YAPACAĞIMIZ SIRA
A. Firebase App Check > Web uygulaması > reCAPTCHA Enterprise kaydı.
B. Oluşan genel site key değerini security-config.js içindeki appCheckSiteKey alanına yazma.
C. Önce metrikleri izleme, sonra Authentication/Firestore/Functions için App Check enforcement açma.
D. Cloud Functions için Blaze planını etkinleştirme.
E. OPENAI_API_KEY değerini Firebase Secret Manager'a ekleme ve functions klasörünü dağıtma.
F. AI çalışınca security-config.js içindeki aiEnabled değerini true yapma.
G. En güçlü içerik koruması için kelime ve hikâye JSON’larını Hosting’den kaldırıp App Check + giriş isteyen backend/Cloud Storage katmanına taşıma. İçerik küçük parçalar hâlinde verilir, hız sınırı ve kullanım kaydı uygulanır.

SALDIRI VE KÖTÜYE KULLANIM İÇİN EK ÖNERİLER
- Firebase bütçe uyarıları ve Cloud Functions maxInstances sınırı.
- App Check enforcement.
- Firestore kurallarını test ederek dağıtma.
- AI/okuma endpointlerinde kullanıcı başına kota.
- Şüpheli trafik için Cloud Logging uyarıları.
- Gizli anahtarları yalnızca Secret Manager’da tutma; GitHub’a veya security-config.js içine koymama.
- JSON anahtarlarını ve servis hesabı dosyalarını kimseyle paylaşmama.

security-config.js BAŞLANGIÇ DURUMU
appCheckSiteKey: boş
aiEnabled: false
Bu ayarlar güvenlik kurulumu tamamlanana kadar uygulamanın mevcut çalışan özelliklerini bozmaz.
